SEO页面交付后客户操作手册

给谁看：拿到我们交付的SEO页面后，需要把它上线并让Google收录的客户 预计耗时：第一次部署30-60分钟，每周维护5-10分钟 支持：遇到任何问题直接联系AIWatch团队 适用客户：SkillBoss / QVeris / Coco / InfiniSynapse 及未来所有客户

你拿到了什么

我们交付给你的是一个完整的SEO页面项目，包括：

  1. 代码仓库（GitHub仓库或zip包）
  2. 页面数据（JSON格式，10-200页不等）
  3. 部署配置（vercel.json，开箱即用）
  4. 本手册（你正在看的这份）

接下来你需要做的事情分3类： - 必做的（5件）→ 不做这些Google就不会收录你 - 建议做的（4件）→ 做了能加快收录、提高排名 - 每周维护（3件）→ 每周花5-10分钟保持效果

一、必做的5件事（按顺序）

第1步：把代码部署到Vercel（15分钟）

方式A（推荐）：从GitHub部署

  1. 打开 vercel.com/new
  2. 点击 Import Git Repository
  3. 选择我们给你的GitHub仓库 → 点击 Deploy
  4. 等待1-2分钟构建完成
  5. 你会得到一个临时域名，类似 xxx-xxx.vercel.app

方式B：直接上传代码

  1. 下载我们给的代码包（zip）
  2. 解压到本地
  3. 打开 vercel.com/new
  4. 把解压后的文件夹拖入上传
  5. 点击 Deploy

验证部署成功

打开临时域名，能看到导航站或首页就说明部署成功。

第2步：绑定你的域名（10分钟）

建议用子域名（比如 cases.yourdomain.com 或 seo.yourdomain.com），不要直接绑主域名。理由：子域名独立运营，不影响主站权重，出问题好隔离。

在Vercel中：

  1. 进入项目的 Settings → Domains
  2. 输入你想用的域名（比如 cases.yourdomain.com）
  3. Vercel会显示一条DNS记录要求

在你的域名管理后台（Cloudflare/GoDaddy/Namecheap等）：

  1. 添加这条CNAME记录： 类型: CNAME 名称: cases （或你选的子域名前缀） 目标: cname.vercel-dns.com TTL: Auto
  2. 保存。等待DNS生效（通常5-30分钟）

验证绑定成功

打开你的域名，能看到页面就说明绑定成功。

第3步：验证Google Search Console（10分钟）

这是收录的前提。不做这一步，Google不会主动来爬你的页面。

  1. 打开 search.google.com/search-console
  2. 点击左上角 添加资源
  3. 选择 网域资源，输入你绑定的子域名（比如 cases.yourdomain.com）
  4. Google会给你一条 TXT记录 要求验证
  5. 把这条TXT记录加到你的DNS里（和第2步同一个地方，类型选TXT）
  6. 等待5-10分钟，回到GSC点击 验证

验证成功后会看到：

  • "已验证所有权"的绿色提示
  • 左侧出现完整的菜单（覆盖率、效果、站点地图等）

第4步：提交Sitemap（2分钟）

告诉Google"这里有N个页面，请来爬"。提交后通常24-72小时内开始爬取。

  1. 在GSC左侧菜单点击 站点地图
  2. 在输入框中填入：sitemap.xml（不需要写完整URL）
  3. 点击 提交
  4. 等待几秒钟，状态会从"无法获取"变成"成功"

如果提交失败：

  • 检查 https://你的域名/sitemap.xml 能否打开
  • 如果打不开，联系AIWatch团队

第5步：每天手动请求索引5-10个URL（持续1周）

这是第一周最重要的动作！比等Google自己来快10倍。

每天花5-10分钟做这件事，第一周内把所有页面都手动请求一遍。

操作步骤：

  1. 在GSC顶部搜索框输入页面完整URL，比如： https://cases.yourdomain.com/use-cases/build-influencer-database
  2. 等它分析完（通常10-20秒）
  3. 点击 请求编入索引
  4. 等待提示"已收到URL编入索引请求"
  5. 换下一个URL，重复

注意：

  • 每天有配额限制（约50次/天），不会让你一次提交所有
  • 第一周每天提交10个左右，一周内全部走完一遍
  • 已经被收录的页面不需要再请求

二、建议做的4件事（提高收录率和排名）

建议1：检查robots.txt不要屏蔽AI爬虫

现在不只是Google收录重要，ChatGPT、Perplexity这些AI搜索引擎也会爬你的页面。屏蔽它们等于在AI搜索里"主动消失"。

打开你的主站 yourdomain.com/robots.txt，确认没有以下内容：

# 这些不能有 Disallow，如果有就删掉
User-agent: GPTBot          # OpenAI/ChatGPT
User-agent: ClaudeBot        # Anthropic/Claude
User-agent: PerplexityBot    # Perplexity
User-agent: Google-Extended  # Google Gemini
User-agent: OAI-SearchBot    # OpenAI搜索

如果有 Disallow: / 之类的屏蔽规则，把它们删掉。

建议2：从主站加2-3个内链指向SEO页面

官网的域名权重很值钱，通过内链可以传递给子域名。

在主站的以下位置加链接： - Footer的"Resources"或"Learn"区块：链到导航站（最推荐） - 博客文章里：相关主题时引用具体某个SEO页面 - 产品页底部的"Use Cases"区块：链到具体场景页

每加一条内链都是给Google一个"这个域名内部认可这些SEO页面"的信号。

建议3：在社交媒体分享3-5个页面

不是为了社交流量，是为了给Google"这个页面有人关注"的信号。

在Twitter / LinkedIn / Reddit分享： - 选3-5个最有用的页面 - 每条分享写不同的角度（不要复制粘贴标题） - 加一句你自己的解读或观点 - 用真人账号分享（CEO/创始人/团队成员），不要用品牌账号

发完不需要管效果——目的是给Google一个外部信号。

建议4：请求1-2个真实外链

大罗SEO的ABC信号里A=外链是最重要的。这是大部分客户的最大短板。

低门槛获取外链的途径：

途径 难度 效果
提交到工具目录站（Product Hunt / Indie Hackers / G2） 低 中
Reddit相关subreddit分享高质量页面 低 中
给行业博主寄样品/送账号换评测 中 高
客座文章（在其他博客发文带链回） 高 高
媒体报道（TechCrunch等） 极高 极高

第一个月做前两条就够了。

三、每周维护（每周5-10分钟）

周维护1：看GSC的核心数据

每周一早上花5分钟看3个数据：

  1. 覆盖率：已收录 / 已发现
  2. 健康比例：>50%
  3. 如果<30%且持续，说明内容质量被Google判低，联系AIWatch团队
  4. 效果（点击和展现）
  5. 看哪些查询给你带来了展现
  6. 看哪些页面有展现但点击率低（可能是标题问题）
  7. 新发现的关键词
  8. GSC会告诉你"用户搜了什么进到你页面"
  9. 这是金矿——根据这些词调整内容或新增页面

周维护2：手动请求新页面索引

如果AIWatch团队给你发了新一批页面（比如W2加了20页）： - 每天提交5-10个新URL请求索引 - 持续到所有新页面都进入"已请求索引"状态

周维护3：响应客户在AI搜索里的提问

如果你在ChatGPT / Perplexity里搜你的产品类别词，发现自己被引用了： - 截图记录（这是宝贵的证据） - 发给AIWatch团队，我们会复盘哪些页面被引用，强化同类内容

如果没被引用： - 也告诉我们，我们会调整GEO策略

四、常见问题FAQ

Q1：部署后页面404怎么办？

A：检查 vercel.json 的rewrites配置是否正确。把错误截图发给AIWatch团队。

Q2：Sitemap提交后状态一直是"无法获取"？

A：90%是 sitemap.xml 文件没正确生成。打开 https://你的域名/sitemap.xml 看能不能直接访问。

Q3：手动请求索引后多久会被收录？

A：通常24-72小时。如果一周后还没收录，可能是页面质量问题。

Q4：被收录但排名很低（>50名），正常吗？

A：完全正常。新站头3个月排名波动很大。重点是： - 第1个月：收录率达标 - 第2-3个月：开始出现长尾词排名 - 第4-6个月：核心词排名稳定

不要只看头部大词，看长尾词的总体表现。

Q5：能不能把页面挪到主域名下？

A：可以但不建议第一个月就挪。建议先在子域名跑3个月看数据，验证内容质量后再迁移到主域名。

Q6：可以编辑/修改页面内容吗？

A：可以，但建议告诉AIWatch团队，我们会同步更新数据源，避免下次更新时被覆盖。

Q7：robots.txt里我看到 Disallow: /admin，要删吗？

A：不要删。/admin 这种是合理的屏蔽（屏蔽后台路径）。只删 Disallow: / 这种全站屏蔽，以及屏蔽AI爬虫的那几条。
